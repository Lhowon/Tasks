Introduction

This is a quick reference guide to Tasks. This will help you understand the key concepts and learn how to the the application.

Concepts

Project

A project is an initiative with a name and time left. A project contains zero or more tasks

Task

A task is an atomic unit of work. A task has several attributes:

  • name: one-line summary describing the task, you may want to name related tasks with a common prefix followed by a colon
  • priority: High or Medium (default) or Low (High ones must be completed, Low ones not included in effort subtotals
  • effort: estimate => actual, in days
  • submitter: User who submitted this task
  • assignee: User to whom this task is assigned
  • type: kind of task to support testing workflow - Feature or Bug or Other (default)
  • status: development status - Planned (default) or Active or Done or Risky
  • validation: validation status - Untested (default) or Passed or Failed
  • description: (multi-line) additional information such as Feature details, Bug reproduction steps, or Release Notes

User

A team member with a login name and full name as well as a role (one of Manager, Developer, or Tester). A user logs into the Tasks aplication and are task assignees and submitters

Basic Features

To begin, click on "Add Project" to create your project and type in a name for it. When you select a project in the list it shows its tasks on the right side. You can remove a selected project by clicking on the "Del. Project" button.

Click "Add Task" to create your tasks. Type in the task descriptions and periodically save your work.
