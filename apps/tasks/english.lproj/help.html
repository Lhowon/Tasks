Introduction

tasks directly supports Agile development practices and the eXtreme Programming (XP) Planning Game. One of the main drivers behind the design of tasks was relentless simplicity - it keeps up with the madness of software development. Developers embrace this tool since it is lightweight and focuses them on what is left to do.

As soon as you understand a few basic concepts you should be off and running in tasks, expanding your sphere of comfort as you learn more. A task is something a user does and tasks are grouped into projects. A user submits (or creates) tasks and these are assigned to users. Sets of tasks are allocated to a project (an initiative or a release). The tasks user interface is comprised of the following sections:

  • A dock at the top with some global action buttons
  • A toolbar below that with actions to add/delete projects and tasks and controls for filtering tasks
  • A workspace in the middle where projects are shown in a master list on the left and tasks are shown in the detail list on the right
  • A control bar near the bottom containing controls to manipulate task attributes
  • A status bar at the bottom to display system messages

Basic Features

Projects

To begin, click on "Add Project" to create a project and type in its name. To view a project's tasks, simply click on the project. To rename a project double click on its name and change it. To delete a project select it and then click on the "Del. Project" button.

There are two reserved projects at the top of the projects list called "All Tasks" and "Unallocated Tasks" projects to show all and unallocated tasks respectively. Remember that tasks is a rich client and you must use the Save button in the dock to periodically save your work. Your iterative releases naturally map to tasks projects - for example, if you are following a Rational Unified Process (RUP) or eXtreme Unified Process (XUP) lifecycle, your SmallReleases become projects like "FeatureReleaseOne" and so on.

Tasks

Tasks are grouped by assignee and displayed using various kinds of color coding - the legend is shown in the bottom control bar. To create a task, click on the "Add Task" button and enter the name of the task. After that, you typically set the task's attributes via the radio buttons in the control bar. You may also click on the icon to the far right of the task to get a popup editor and enter additional details about the task. Update a task's name by double clicking on it and using the inline editor. Delete selected tasks using the "Del. Task" button. To reassign a task to another user, simply drag and drop it to the target user if they are visible or use the tak popup editor to change the assignee. To reallocate a task to another project drag and drop it on the target project in the master list. Note: you can select multiple tasks at once and perform delete/reassign/reallocate operations.

When entering a task, try to use the XP principle of FeaturesNotSubsystems - try to have user-visible Features which may need some Other tasks as infrastructure. XP's RefactorMericilessly manifests itself as Bug and Other tasks - pay off your technical debt as you go.

Users

Click on the "Settings" button in the dock to access the User Manager panel. Here you can add/modify/delete users and their roles.

Advanced Features

Grouping/Traceability

Prefix names of related tasks with Use Case or Component names so that these tasks can be found by searching for that prefix. tasks also allows you to trace back to upstream Analysis artifacts via the task description: you can add a link to an external Wiki page or Content Management System (CMS) artifact.

Workflow

tasks supports a simple workflow as shown in the control bar at the bottom. You first plan your tasks, specifying their types and priorities. Then you work on your tasks, noting your status as you go. Finally, you may validate your completed tasks.

Sorting

Sorting in tasks is automatic. Projects are sorted alphabetically. Tasks are sorted by status, validation (if Done), priority, and type in that order. This helps you focus on what you are working on now and what is next.

Filtering

Another useful feature is task filtering - you may do that in three ways:

  • Click on the "Filter Tasks" button and check/uncheck the task attributes you want filter on
  • Specify task assignees (comma or space separated login names)
  • Search using task IDs (each prepended by '#' and comma or space separated) or a string matching any part of task names in the search bar

Filtering results are displayed as soon as you stop typing. If an attribute filter is set you will see it indicated by a special icon next to the "Filter Tasks" button.

Load Balancing

To use this feature, you must specify the time left in a project as well as the effort for tasks. tasks will automatically total the efforts for each assignee and indicate their loading via color coding. A "?" is appended for range-based subtotals including tasks with unspecified efforts.

Data Exchange

tasks allows data to be exchanged via a simple import/export mechanism. Use the "Import" button in the toolbar to bring data into the application. Use the "Export" button to output tasks data to another installation of the application or some other system.

Usage Tips

To get the maximum value from using tasks, consider the following usage tips:

  • You can manage by exception by filtering to only see Risky or Failed tasks
  • Hotkeys are available for many common actions: Save = Ctrl-S, New Task = Ctrl-T, New Project = Ctrl-P
  • Right clicking on projects and tasks provides a context menu with common actions
  • If you are almost finished with a task and have a small final portion left, split up the task and mark the completed portion Done - this will give you credit for the completed portion and avoid the overloaded assignee label!
  • Use the task submitter to indicate your collaborator - the person you will consult with

Power User Features

Finally, tasks has some great hidden features to avoid confusing regular users. As you get more comfortable with tasks, ty using some of these shortcuts.

  • Type in a project's time left inline by typing it within curly braces
  • Type in a tasks' properties/attributes inline using the same syntax for Import/Export
  • Reassign a task to another user by typing in their login name within square brackets inline at the end of a task
  • Use the Import panel to type in a bunch of tasks at once - basically use it as a simple text editor

Reference

This section goes into details about the various entities used in the application.

User

A team member with a login name and full name as well as a role (one of Manager, Developer, or Tester). A user logs into the tasks application using an optional password and are task assignees and submitters.

Task

An atomic unit of work with several attributes:

  • name: one-line summary describing the task, you may want to name related tasks with a common prefix followed by a colon
  • priority: High or Medium (default) or Low (High ones must be completed, Low ones not included in effort subtotals
  • effort: optional estimate => actual, appended with 'd' (days) or 'h' (hours)
  • project: project this task is allocated to, if any
  • submitter: user who submitted this task, if any
  • assignee: user to whom this task is assigned, if any
  • type: kind of task - Feature or Bug or Other (default)
  • status: development status - Planned (default) or Active or Done or Risky
  • validation: validation status - Untested (default) or Passed or Failed
  • description: optional (multi-line) additional information such as Feature details, Bug reproduction steps, or Release Notes

Project

An initiative with a name, optional time left, zero or more tasks, and an optional description.
