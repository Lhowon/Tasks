Introduction

Tasks is built around a few simple concepts. A "task" is something a "user" does and tasks are grouped into "projects." A user "submits" (or creates) tasks and these are "assigned" to users. Sets of tasks are "allocated" to a project. A project may be an initiative or a release. The Tasks user interface is comprised of the following sections:

  • A "dock" at the top with some global action buttons
  • A "toolbar" below that with actions to add/delete/filter projects and tasks
  • A "workspace" in the middle where projects are shown in a "master" list on the left and tasks are shown in the "detail" list on the right
  • A "control bar" near the bottom containing controls to manipulate task attributes
  • A "status bar" at the bottom to display system messages

Getting Started

Projects

To begin, click on "Add Project" to create a project and type in its name. To view a project's tasks, simply click on the project. To rename a project double click on its name and change it. To delete a project select it and then click on the "Del. Project" button.

Tasks

Tasks are grouped by assignee and displayed using various kinds of color coding - the legend is shown in the bottom control bar. To create a task, click on the "Add Task" button and enter the name of the task. After that, you typically set the task's attributes via the radio buttons in the control bar. You may also click on the icon to the far right of the task to get a popup editor and enter additional details about the task. Update a task's name by double clicking on it and using the inline editor. Delete a task using the "Del. Task" on the selected task. To reassign a task to another user, simply drag and drop it to the target user if they are visible or use the tak popup editor to change the assignee. To reallocate a task to another project drag and drop it on the target project in the master list. Note: you can select multiple tasks at once and perform delete/reassign/reallocate operations.

Users

Click on the "Settings" button in the dock to acsess the User Manager panel. Here you can add/modify/delete users and their roles.

Advanced Features

Workflow

Tasks supports a simple workflow as shown in the control bar at the bottom. You first decide on the type of task, then specify the priority, then note your development status, and finally the validation status. Unless a task is done, you cannot validate it.

Sorting

Sorting in Tasks is automatic. Projects are sorted alphabetically. Tasks are sorted by status, validation (if Done), priority, and type in that order. This helps you keep focus on what you are working on and what is next.

Filtering

Another useful feature is task filtering - you may do that in three ways:

  • Click on the "Filter Tasks" button and check/uncheck the task attributes you want filter on
  • Specify task assignees (comma or space separated login names)
  • Search using task IDs (each prepended by '#' and comma or space separated) or a string matching any part of task names in the search bar

Filtering happens as you type - you will see the results immediately. If an attribute filter is set you will see it indicated by a special icon next to the "Finter Tasks" button.

There are two virtual projects at the top of the projects list called "All Tasks" and "Unallocated Tasks" projects to show all and unallocated tasks respectively. Remember that Tasks is a rich client and you must use the Save button in the dock to periodically save your work.

Load Balancing

To use this feature, you can specify the "time left" in a project as well as the "effort" for tasks. The system will automatically total the efforts for each assignee and indicate their loading via color coding. A "?" is appended for range-based subtotals including tasks with unspecified efforts.

Data Exchange

Tasks allows data to be exchanged via a simple import/export mechanism. Use the "Import" button in the toolbar to bring data into the application. Use the "Export" button to output Tasks data to another Tasks application or some other system.

Hidden Features

Tasks has some great power-user features that are hidden to avoid confusing regular users. As you get more comfortable with Tasks, try to learn these shortcuts to save yourself a lot of time.

  • Right clicking on project and tasks provides a context menu
  • Hotkeys are available for most of the common actions
  • You can type in a project's "time left" inline by typing it within curly braces
  • You can type in a tasks' attributes using the same syntax for Import/Export!

Reference Guide

This section goes into details about the various entities used in the application.

Project

A project is an initiative with a name and time left. A project contains zero or more tasks

Task

A task is an atomic unit of work. A task has several attributes:

  • name: one-line summary describing the task, you may want to name related tasks with a common prefix followed by a colon
  • priority: High or Medium (default) or Low (High ones must be completed, Low ones not included in effort subtotals
  • effort: estimate => actual, in days
  • submitter: User who submitted this task
  • assignee: User to whom this task is assigned
  • type: kind of task to support testing workflow - Feature or Bug or Other (default)
  • status: development status - Planned (default) or Active or Done or Risky
  • validation: validation status - Untested (default) or Passed or Failed
  • description: (multi-line) additional information such as Feature details, Bug reproduction steps, or Release Notes

User

A team member with a login name and full name as well as a role (one of Manager, Developer, or Tester). A user logs into the Tasks aplication and are task assignees and submitters
